# Prompt — GitHub Repository and Reproducibility

## Copy-paste prompt

You are an open-source maintainer preparing a hackathon repository for judges.

Review and improve the repository for **Aircraft Maintenance Decision Copilot**.

## Required repository quality

- Clear project title and one-sentence value proposition.
- Architecture diagram.
- Demo screenshot or GIF.
- Video link placeholder.
- Deployed app link placeholder.
- Quickstart under ten commands.
- Docker Compose setup.
- Seeded synthetic demo data.
- `.env.example`.
- License.
- Contribution notes.
- Test instructions.
- Evaluation instructions.
- Security and limitations section.
- Partner technology acknowledgements.
- No secrets.
- No large generated files.
- Meaningful commit history where possible.

## Required README structure

```markdown
# Project title
## Problem
## Solution
## Demo
## Architecture
## Agent workflow
## Technology stack
## Quickstart
## Configuration
## Demo scenario
## Evaluation
## Security
## Limitations
## Future work
## Team
## Acknowledgements
```

## Your task

1. Audit the current repository.
2. Produce a prioritized issue list.
3. Rewrite `README.md`.
4. Create issue templates.
5. Create pull request template.
6. Create GitHub Actions for lint, test, and secret scan.
7. Verify all commands are consistent.
8. Add a release checklist.

Do not claim tests, deployments, metrics, or integrations that have not been verified.
