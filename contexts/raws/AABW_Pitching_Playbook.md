## Slide 02 — Win the Room in Five Minutes

**Agentic AI Build Week**

# Win the room in five minutes

A practical Round 1 pitch workshop.

- **5-minute pitch**
- **2-minute Q&A**
- **20 min sharing**
- **10 min discussion**

---

## Slide 03 — Your Pitch Has One Job

### Workshop outcome

Make judges:

| Goal           | What judges should understand |
| -------------- | ----------------------------- |
| **UNDERSTAND** | the problem                   |
| **SEE**        | the agent work                |
| **BELIEVE**    | the outcome                   |
| **TRUST**      | the team                      |

> **Clarity beats coverage.**

---

## Slide 04 — Design the Story Around the Rubric

### The judge’s lens

|   # | Criterion                | Question to answer                         |
| --: | ------------------------ | ------------------------------------------ |
|  01 | **Agentic AI use**       | Does it plan, reason, use tools and act?   |
|  02 | **Problem / track fit**  | Does the team understand the real problem? |
|  03 | **Technical execution**  | Is it functional and credible?             |
|  04 | **Impact / usefulness**  | What becomes measurably better?            |
|  05 | **Creativity**           | Why is this approach different?            |
|  06 | **Pitch / demo clarity** | Can I follow and remember it?              |

---

## Slide 05 — Give Every Second a Purpose

### The five-minute map

| Time | Section         | Purpose                 |
| ---: | --------------- | ----------------------- |
|  40s | **Identity**    | Team + hook             |
|  40s | **Problem**     | Insight, not repetition |
|  60s | **Solution**    | Agentic workflow        |
|  45s | **Credibility** | Why it works            |
|  45s | **Impact**      | Proof or early evidence |
|  60s | **Demo**        | Outcome in action       |
|  10s | **Close**       | Final takeaway          |

> **Rehearse to 4:45. Keep 15 seconds of safety.**

---

## Slide 06 — Open with Identity, Relevance and a Hook

**Timing:** `0:00–0:40`

### Identity

- **We are:** `[Team]`
- **We built:** `[Project]`

### Promise statement

> For `[target user]`, we deliver **[valuable outcome]** by using an AI agent to **[distinctive action]**.

### Credibility line

> Domain expertise × AI engineering × Product strength

---

## Slide 07 — Show Insight, Not a Copy of the Brief

**Timing:** `0:40–1:20`

| Element        | What to explain              |
| -------------- | ---------------------------- |
| **Who**        | A specific user              |
| **Goal**       | What they need to achieve    |
| **Friction**   | Where the workflow breaks    |
| **Root cause** | Why current options fail     |
| **Evidence**   | What you learned or measured |

> Say **“We discovered…”**, not **“The brief says…”**

> One insight is worth more than five generic statistics.

---

## Slide 08 — Show the Agent Doing the Work

**Timing:** `1:20–2:20`

```text
GOAL → PLAN → TOOLS → ACT → VERIFY
```

|            Step | Agent behavior                   |
| --------------: | -------------------------------- |
|   01 — **Goal** | Understands the user’s objective |
|   02 — **Plan** | Breaks work into steps           |
|  03 — **Tools** | Selects data and systems         |
|    04 — **Act** | Executes the workflow            |
| 05 — **Verify** | Checks results or escalates      |

> If judges cannot see the decision and action, they may see only a chatbot.

---

## Slide 09 — Translate Every Feature into User Value

### Solution clarity

| Don’t stop at the feature | Explain the user value: “So what?”                   |
| ------------------------- | ---------------------------------------------------- |
| **Multiple agents**       | Specialists divide and validate the workflow         |
| **API integrations**      | The task is completed across tools users already use |
| **Automated reports**     | Scattered inputs become a decision in minutes        |

Complete the sentence:

> **“This matters because…”**

---

## Slide 10 — Prove It Works Without Teaching the Stack

**Timing:** `2:20–3:05`

Show only the choices that build belief.

| Principle      | Evidence to show          |
| -------------- | ------------------------- |
| **Real**       | End-to-end workflow       |
| **Reliable**   | Validation + recovery     |
| **Controlled** | Permissions + approval    |
| **Evaluated**  | Representative test cases |

> **Architecture is evidence—not the main character.**

---

## Slide 11 — Use the Strongest Evidence You Actually Have

**Timing:** `3:05–3:50`

Evidence hierarchy, from strongest to weakest:

| Level | Evidence type     | Meaning                        |
| ----: | ----------------- | ------------------------------ |
|     4 | **Pilot results** | Measured with target users     |
|     3 | **User tests**    | Observed tasks + feedback      |
|     2 | **Benchmarks**    | Representative scenarios       |
|     1 | **Assumptions**   | Estimated and clearly labelled |

Measure:

- Time
- Cost
- Errors
- Completion
- Access
- Satisfaction

---

## Slide 12 — A One-Minute Demo Needs a Storyline

**Timing:** `3:50–4:50`

| Demo time | Stage          | What to show             |
| --------: | -------------- | ------------------------ |
|     0–10s | **Goal**       | What the user needs      |
|    10–20s | **Trigger**    | What starts the workflow |
|    20–45s | **Agent acts** | Plan • tools • decisions |
|    45–55s | **Outcome**    | What the user receives   |
|    55–60s | **Proof**      | One memorable result     |

> **Skip login, setup and navigation. End on the user outcome.**

---

## Slide 13 — Six Slides Can Carry the Whole Pitch

### Recommended main deck

1. **Team + promise**
2. **Problem insight**
3. **Agentic workflow**
4. **Why it wins**
5. **Evidence + impact**
6. **Demo + close**

> The deck supports the story. It should never compete with the speaker.

---

## Slide 14 — Collect the Questions Before Answering

### The two-minute Q&A

Ask:

> “Could we collect the judges’ questions first?”

Suggested timing:

- **0:20** — Collect questions
- **1:20** — Answer questions

Process:

```text
Note → Group → Answer in order
```

---

## Slide 15 — Answering Well

# Direct. Supported. Honest.

| Step | Method      | Guidance                  |
| ---: | ----------- | ------------------------- |
|    1 | **Answer**  | Lead with the conclusion. |
|    2 | **Support** | Give one fact or example. |
|    3 | **Connect** | Return to user value.     |

> Not sure? Say what is unvalidated, what you know, and how you will test it.

**Target:** 15–25 seconds per answer.

## Slide 16 — Prepare the Pitch Judges Can Score

### Your next move

1. **Write** one sentence per rubric criterion.
2. **Cut** anything that does not build belief.
3. **Record** a 60-second outcome demo.
4. **Rehearse** to 4:45 with hard questions.

> Questions—or which part of your pitch should we pressure-test?

## Slide 19 — Rehearse the Questions That Test Credibility

### Appendix: Q&A bank

| Area           | Questions                                        |
| -------------- | ------------------------------------------------ |
| **Agent**      | Why an agent? What decisions? What tools?        |
| **Problem**    | Who validated it? What alternatives exist?       |
| **Tech**       | What is real? What fails? How is data protected? |
| **Impact**     | Where do the numbers come from? Who tested it?   |
| **Difference** | Why can’t a general assistant do this?           |

## Slide 20 — Final Rehearsal Checklist

### Appendix: Before submission

- [ ] Names + track are visible
- [ ] Problem insight has evidence
- [ ] Agent planning and tool use are explicit
- [ ] Technical claims match what works
- [ ] Impact is measured or clearly labelled
- [ ] Demo is ≤ 60 seconds
- [ ] Demo ends on the outcome
- [ ] Pitch finishes by 4:45
- [ ] Appendix is numbered
- [ ] Q&A owners are assigned
